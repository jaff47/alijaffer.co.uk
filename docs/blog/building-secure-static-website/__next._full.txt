f:T9c2,<p>This post explains how I built and deployed this website using modern web technologies, automated workflows, and security best practices.</p>
<h2>Cost Breakdown</h2>
<p>The entire setup is remarkably cost-effective. The only expense is the domain registration (approximately £10-15 per year). Everything else is free:</p>
<ul>
<li>GitHub Pages hosting - Free</li>
<li>SSL/TLS certificate - Free (provided by GitHub)</li>
<li>GitHub Actions CI/CD - Free for public repositories</li>
<li>Private repository - Free with GitHub account</li>
</ul>
<p>This makes it an excellent solution for personal websites and portfolios without ongoing hosting costs.</p>
<h2>Architecture Overview</h2>
<p>The site uses a dual-repository approach: a private development repository for source code and a public repository for hosting. This separation ensures source code remains private while the built site is publicly accessible.</p>
<p><img src="/devops-pipeline.svg" alt="DevOps Pipeline Diagram"></p>
<h2>Technology Stack</h2>
<ul>
<li>Next.js 16 with static export for optimal performance</li>
<li>TypeScript for type safety</li>
<li>Tailwind CSS for styling</li>
<li>Jest and React Testing Library for automated testing</li>
<li>GitHub Actions for CI/CD</li>
<li>GitHub Pages for hosting</li>
</ul>
<h2>Deployment Pipeline</h2>
<p>Every push to the main branch triggers an automated workflow that runs tests, scans for secrets using TruffleHog, builds the static site, and deploys to the public repository. This ensures only tested, secure code reaches production.</p>
<h2>Security Measures</h2>
<ul>
<li>Secret scanning prevents credential leaks</li>
<li>Automated security audits on dependencies</li>
<li>External links use rel=&quot;noopener noreferrer&quot;</li>
<li>Custom domain with HTTPS enabled</li>
<li>Source code isolated in private repository</li>
</ul>
<h2>Custom Domain Setup</h2>
<p>The site uses a custom domain configured through DNS A records pointing to GitHub Pages servers, with a CNAME file ensuring the domain persists across deployments.</p>
<h2>Key Takeaways</h2>
<p>Static site generation offers excellent performance and security. Automated testing and deployment pipelines ensure reliability. Separating source code from deployment artifacts maintains security while enabling public hosting.</p>
<p>The entire setup demonstrates how modern DevOps practices can be applied to personal projects, creating a professional, secure, cost-effective and maintainable web presence.</p>
